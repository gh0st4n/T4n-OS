Installation Via Chroot(x86/x86_64)

Panduan ini menjelaskan secara detail proses instalasi Void secara manual melalui chroot pada sebuah sistem. Arsitektur x86, x86_64, atau aarch64. Diasumsikan Anda sudah familiar dengan arsitektur tersebut. dengan Linux, tetapi belum tentu dengan menginstal sistem Linux melalui chroot. Panduan ini dapat digunakan untuk membuat pengaturan "tipikal", menggunakan satu partisi pada satu disk SATA/IDE/USB. Setiap langkah dapat dimodifikasi untuk menciptakan solusi yang kurang umum. pengaturan, seperti Full Disk Encryption.

Void menyediakan dua opsi untuk memulai instalasi baru. XBPS Metode ini menggunakan XBPS Package Manager yang berjalan pada sebuah host. Sistem operasi untuk menginstal sistem dasar. Metode ROOTFS menginstal sistem dasar dengan mengekstrak file tarball ROOTFS.

Metode XBPS mensyaratkan bahwa sistem operasi host telah menginstal XBPS. Ini mungkin merupakan instalasi Void yang sudah ada, sebuah pertunjukan langsung resmi citra , atau instalasi Linux apa pun yang menjalankan secara statis XBPS terhubung.

Metode ROOTFS hanya memerlukan sistem operasi host yang dapat masuk ke Linux chroot dan itu memiliki tar dan xz terpasang. Metode ini mungkin lebih disukai jika Anda ingin menginstal Void menggunakan distribusi Linux yang berbeda.

Prepare Filesystems

Partisi Disk Anda dan format menggunakan mke2fs, mkfs.xfs, mkfs.btrfs atau alat apa pun yang tersedia diperlukan untuk sistem file pilihan Anda.

cfdisk dan fdisk tersedia pada gambar langsung untuk partisi, tetapi Anda mungkin ingin menggunakan gdisk (dari paket gptfdisk) atau parted sebagai gantinya.

Untuk sistem booting UEFI, pastikan untuk membuat Partisi Sistem EFI (ESP). ESP harus memiliki tipe partisi "Sistem EFI" (kode EF00) dan menjadi diformat sebagai FAT32 menggunakan mkfs.vfat.

Jika Anda tidak yakin partisi apa yang harus dibuat, buat partisi 1GB dengan tipe "EFI" Sistem" (kode EF00), kemudian buat partisi kedua dengan tipe "Sistem Berkas Linux" (kode 8300) menggunakan sisa ruang penyimpanan di hard drive.

Format partisi-partisi ini masing-masing sebagai FAT32 dan ext4:

# mkfs.vfat /dev/sda1

# mkfs.ext4 /dev/sda2

Create a New Root and Mount a File System

Panduan ini mengasumsikan bahwa sistem file root yang baru telah terpasang pada /mnt Anda mungkin ingin memasangnya di tempat lain.

Jika menggunakan UEFI, pasang Partisi Sistem EFI sebagai /mnt/boot/efi.

Sebagai contoh, jika /dev/sda2 akan dipasang sebagai / Dan dev/sda1 adalah EFI Partisi Sistem:

# mount /dev/sda2 /mnt/

# mkdir -p /mnt/boot/efi/2

# mount /dev/sda1 /mnt/boot/efi/

Inisialisasi ruang swap, jika diinginkan, menggunakan mkswap, dan mengaktifkannya dengan swapon.

Basic Installation

Ikuti hanya salah satu dari dua subbagian berikut.

Jika menggunakan arsitektur aarch64, maka perlu menginstal paket kernel tambahan. base-system. Misalnya, linux adalah paket kernel yang mengarah ke Kernel stabil terbaru yang dikemas oleh Void.

The XBPS Method

Pilih mirror dan gunakan URL yang sesuai untuk Jenis sistem yang ingin Anda instal. Untuk mempermudah, simpan URL ini ke shell. variabel. Instalasi glibc, misalnya, akan menggunakan:

# REPO=https://repo-default.voidlinux.org/current

XBPS juga perlu mengetahui arsitektur apa yang sedang diinstal. Opsi yang tersedia adalah x86_64, x86_64-musl, i686 untuk komputer arsitektur PC dan aarch64. Misalnya:

# ARCH=x86_64

Arsitektur ini harus kompatibel dengan sistem operasi Anda saat ini, tetapi Tidak harus sama. Jika host Anda menjalankan sistem operasi x86_64 sistem, salah satu dari tiga arsitektur dapat diinstal (apakah host tersebut musl atau glibc), tetapi host i686 hanya dapat menginstal distribusi i686.

Salin kunci RSA dari media instalasi ke direktori root target:

# mkdir -p /mnt/var/db/xbps/keys

# cp /var/db/xbps/keys/* /mnt/var/db/xbps/keys/

Gunakan xbps-install untuk melakukan bootstrap pemasangan dengan memasang base-system meta package:

# XBPS_ARCH=$ARCH xbps-install -S -r /mnt -R "$REPO" base-system

The ROOTFS Method

Unduh ROOTFS tarball sesuai dengan arsitektur Anda.

Ekstrak file tarball ke dalam sistem file yang baru dikonfigurasi:

# tar xvf void-<...>-ROOTFS.tar.xz -C /mnt

Citra ROOTFS umumnya berisi perangkat lunak yang sudah usang, karena merupakan cuplikan dari waktu pembuatannya, dan tidak dilengkapi dengan kelengkapan base-system. Perbarui pengelola paket dan instal base-system:

# xbps-install -r /mnt -Su xbps

# xbps-install -r /mnt -u

# xbps-install -r /mnt base-system

# xbps-remove -r /mnt -R base-container-full

Konfigurasi

Bagian selanjutnya dari panduan ini berlaku sama untuk instalasi XBPS dan ROOTFS. metode.

File fstab dapat secara otomatis dihasilkan dari sistem file yang saat ini terpasang menggunakan xgenfstab (dari xtools).

# xgenfstab -U /mnt > /mnt/etc/fstab

Memasuki Chroot

xchroot (dari xtools) dapat digunakan untuk Siapkan dan masuki chroot. Atau, ini bisa dilakukan secara manual .

# xchroot /mnt /bin/bash

Konfigurasi Instalasi

Tentukan nama host di /etc/hostname Telusuri opsi-opsi yang tersedia /etc/rc.conf Jika memasang glibc distribusi, penyuntingan /etc/default/libc-locales, menghapus komentar yang diinginkan locale.

nvi(1) tersedia di chroot, tetapi Anda Anda mungkin ingin menginstal editor teks pilihan Anda saat ini.

Untuk build glibc, hasilkan file lokal dengan:

[xchroot /mnt] # xbps-reconfigure -f glibc-locales

Tetapkan Kata Sandi Root

Konfigurasikan setidaknya satu akun super user . Akun pengguna lain dapat dikonfigurasi nanti, tetapi setidaknya harus ada akun root. kata sandi, atau akun pengguna baru dengan sudo hak istimewa.

Untuk mengatur kata sandi root, jalankan perintah berikut:

[xchroot /mnt] # passwd

Aktifkan layanan

Menggunakan instalasi grub untuk menginstal GRUB ke disk boot Anda.

Pada komputer BIOS , instal paket tersebut. grub lalu jalankan grub-install /dev/sdX, Di mana /dev/sdX adalah drive (bukan partisi) yang ingin Anda Instal GRUB juga. Misalnya:

[xchroot /mnt] # xbps-install -S grub

[xchroot /mnt] # grub-install /dev/sda

Pada komputer UEFI , instal salah satu dari berikut ini: grub-x86_64-efi, grub-i386-efi atau grub-arm64-efi, tergantung pada arsitektur Anda. Kemudian jalankan grub-install dengan yang benar --target argumen untuk arsitektur Anda ( x86_64-efi, i386-efi, atau arm64-efi), dan secara opsional menentukan label bootloader (label ini dapat berupa (digunakan oleh firmware komputer Anda saat memilih perangkat boot secara manual):

[xchroot /mnt] # xbps-install -S grub-x86_64-efi

[xchroot /mnt] # grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id="T4nOS"

Panduan pemecahan masalah instalasi GRUB

Mungkin perlu untuk memasang efivarfs sistem berkas.

[xchroot /mnt] # mount -t efivarfs none /sys/firmware/efi/efivars

Jika variabel EFI masih belum tersedia, tambahkan opsi tersebut. --no-nvram ke grub-install memerintah.

Instalasi pada media yang dapat dilepas atau sistem UEFI yang tidak kompatibel

Sayangnya, tidak semua sistem memiliki UEFI yang sepenuhnya sesuai standar. implementasi. Dalam beberapa kasus, perlu untuk "mengakali" firmware agar melakukan booting dengan menggunakan lokasi fallback default untuk bootloader, bukan yang khusus. Dalam hal itu, atau jika menginstal ke disk yang dapat dilepas (seperti USB), tambahkan opsi --removableke grub-install memerintah.

Alternatifnya, gunakan mkdir(1) untuk membuat /boot/efi/EFI/bootdirektori dan salin file eksekusi GRUB yang sudah terinstal, biasanya terletak di /boot/efi/EFI/Void/grubx64.efi(lokasinya dapat ditemukan dengan menggunakan efibootmgr(8) ), ke dalam folder baru:

[xchroot /mnt] # mkdir -p /boot/efi/EFI/boot

[xchroot /mnt] # cp /boot/efi/EFI/Void/grubx64.efi /boot/efi/EFI/boot/bootx64.efi

Finalisasi

Gunakan xbps-reconfigure untuk Pastikan semua paket yang terpasang telah dikonfigurasi dengan benar:

[xchroot /mnt] # xbps-reconfigure -fa

Ini akan membuat dracut(8) menghasilkan sebuah initramfs, dan akan membuat GRUB menghasilkan konfigurasi yang berfungsi.

Pada tahap ini, instalasi telah selesai. Keluar dari chroot dan mulai ulang komputer Anda. komputer:

[xchroot /mnt] # exit

# umount -R /mnt

# shutdown -r now

Setelah melakukan booting ke instalasi Void Anda untuk pertama kalinya, lakukan pemeriksaan sistem memperbarui.
