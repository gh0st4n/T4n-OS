Partition Notes

Pemartisian untuk distribusi Linux modern umumnya sangat sederhana, namun Pengenalan GPT dan booting UEFI memang membawa kompleksitas baru pada proses. Saat membuat tabel partisi baru, Anda akan memerlukan partisi untuk Sistem file root, beserta partisi swap dan mungkin partisi lain. atau dua untuk mempermudah proses booting, jika diperlukan.

Perhatikan bahwa jika disk sudah diinisialisasi, bagian atas cfdisk Layar akan menampilkan tata letak partisi yang sudah ada: Label: dosuntuk MBR skema, Label: gptuntuk skema GPT. Jika Anda hanya ingin menghapus partisi. Sebelum memulai penginstal, gunakan tabel berikut: wipefs(8)Jika tidak, Anda bisa menjalankan. cfdisk(8)secara manual dengan -zopsi untuk memulai dengan disk yang belum diinisialisasi tata letak; cfdiskakan meminta Anda untuk memilih jenis label sebelum melanjutkan ke layar utama.

Bagian-bagian berikut akan menjelaskan secara rinci opsi-opsi untuk konfigurasi partisi.

Catatan sistem BIOS

Disarankan untuk membuat tabel partisi MBR jika Anda menggunakan BIOS. Sistem boot. Ini akan membatasi jumlah partisi yang dapat Anda buat menjadi empat.

Dimungkinkan untuk menggunakan tabel partisi GPT pada sistem BIOS, tetapi GRUB akan Membutuhkan partisi khusus agar dapat melakukan booting dengan benar. Partisi ini harus berada di... di awal disk Anda dan memiliki ukuran 1MB, dengan tipe BIOS boot(GUID 21686148-6449-6E6F-744E-656564454649). Jangan membuat sistem file apa pun di dalamnya. GRUB Kemudian seharusnya akan terinstal dengan sukses.

Catatan sistem UEFI

Pengguna UEFI disarankan untuk membuat tabel partisi GPT. Booting UEFI dengan GRUB juga memerlukan partisi khusus dengan tipe tertentu. EFI Systemdengan vfat sistem berkas terpasang di /boot/efiUkuran yang wajar untuk partisi ini bisa jadi... antara 200MB dan 1GB. Dengan pengaturan partisi ini selama live image. Setelah instalasi, installer seharusnya berhasil mengatur bootloader. secara otomatis.

Swap Partition

Partisi swap sebenarnya tidak wajib, tetapi direkomendasikan untuk sistem dengan konsumsi sumber daya yang rendah. RAM. Jika Anda ingin menggunakan hibernasi, Anda memerlukan partisi swap. Tabel berikut berisi rekomendasi untuk ukuran partisi swap.

RAM Sistem... Swap Ram Yang Direkomendasikan... Gunakan Swap Space jika menggunakan hibernasi...
< 2 GB 2x jumlah RAM 3 kali lipat jumlah RAM
2-8 GB Sama dengan jumlah RAM 2x jumlah RAM
8-64 GB Minimal 4 GB 1,5 kali jumlah RAM
64 GB Minimal 4 GB Hibernasi tidak disarankan

Partisi boot (opsional)

Pada sebagian besar sistem modern, terdapat panel terpisah. /boot partisi tidak lagi diperlukan untuk Pastikan booting dengan benar. Jika Anda memilih untuk menggunakannya, perhatikan bahwa T4n OS tidak menghapus yang lama. kernel setelah pembaruan secara default dan juga bahwa kernel cenderung meningkat ukurannya berbeda-beda di setiap versi baru, jadi rencanakan sesuai kebutuhan (misalnya, /bootdengan satu Linux 5.x x86_64kernel dan GRUB membutuhkan sekitar 60MB).

Partisi Lainnya

Tidak masalah untuk menginstal sistem Anda hanya dengan partisi root yang besar, tetapi Anda mungkin Buat partisi lain jika Anda mau. Salah satu tambahan yang bermanfaat adalah partisi terpisah. partisi untuk Anda /homedirektori. Dengan cara ini, jika Anda perlu menginstal ulang Void (atau (distribusi lain) Anda dapat menyimpan data dan file konfigurasi di direktori home Anda. direktori untuk sistem baru Anda.
