History T4n OS

Awal Mula

T4n OS lahir dari kebutuhan sederhana tapi nyata: Void Linux itu cepat, bersih, dan minimal.

Masalah Void Linux

Beberapa masalah utama saat menggunakan Void Linux:

  • Banyak Software Tida Tersedia di repo Official
  • xbps-src kuat tapi ribet untuk penggunaan harian
  • Tidak ada ekosistem seperti AUR
  • Runit powerful, tapi setup default kurang fleksibel untuk berbagai use-case

Runit & Fleksibilitas Init

Void identik dengan runit.

T4n OS memilih tidak memaksa.

  • runit → default
  • systemd → tetap opsi, jika user mau

T4n OS fokus ke :

  • Struktur service jelas
  • Mudah dimodifikasi
  • Desktop(Utama) dan Server(Coming Soon)

Status Saat Ini

  • Eksperimental
  • Berkembang Perlahan
  • Fokus ke kualitas & kontrol
Penutup: T4n OS adalah OS Proyek Mandiri & Gabut.
