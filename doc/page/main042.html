Network

Konfigurasi jaringan di Void Linux dapat dilakukan dengan beberapa cara. Secara default, cara ini memungkinkan konfigurasi jaringan di Void Linux. instalasi dilengkapi dengan service dhcpcd diaktifkan.

Interface

Versi udev yang lebih baru tidak lagi menggunakan skema penamaan Linux tradisional untuk antarmuka ( eth0, eth1, wlan0, ...).

Perilaku ini dapat dibalik dengan menambahkan net.ifnames=0 ke Kernel-CMD

Konfigurasi Statis

Cara sederhana untuk mengkonfigurasi jaringan statis saat boot adalah dengan menambahkan yang diperlukan. perintah ip ke /etc/rc.local berkas: 

ip link set dev eth0 up
ip addr add 192.168.1.2/24 brd + dev eth0
ip route add default via 192.168.1.1

Bridge Interface

Untuk mengkonfigurasi antarmuka bridge saat boot, /etc/rc.local berkas dapat digunakan untuk jalankan perintah ip untuk menambahkan bridge br0 Dan tetapkan sebagai master untuk eth0 antarmuka sebagai contoh: 

ip link add name br0 type bridge
ip link set eth0 master br0
ip link set eth0 up

dhcpcd

Untuk menjalankan dhcpcd pada semua antarmuka, aktifkan itu service dhcpcd.

Untuk run dhcpcd hanya pada antarmuka tertentu, salin service dhcpcd-eth0 dan Ubah agar sesuai dengan antarmuka Anda: 

$ ip link show
1: lo:  mtu 65536 qdisc noqueue state UNKNOWN mode DEFAULT group default qlen 1
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
2: enp3s0:  mtu 1500 qdisc pfifo_fast state UP mode DEFAULT group default qlen 1000
        link/ether ff:ff:ff:ff:ff:ff brd ff:ff:ff:f
# cp -R /etc/sv/dhcpcd-eth0 /etc/sv/dhcpcd-enp3s0
# ln -nsf /run/runit/supervise.dhcpcd-enp3s0 /etc/sv/dhcpcd-enp3s0/supervise"
# sed -i 's/eth0/enp3s0/' /etc/sv/dhcpcd-enp3s0/run
# ln -s /etc/sv/dhcpcd-enp3s0 /var/service/

Untuk informasi lebih lanjut tentang konfigurasi dhcpcd, merujuk pada dhcpcd.conf

Nirkabel

Sebelum menggunakan jaringan nirkabel, gunakan rfkill untuk memeriksa apakah yang relevan Antarmuka diblokir secara lunak atau keras.

Void menyediakan beberapa cara untuk terhubung ke jaringan nirkabel:

  • wpa_supplicant
  • NetworkManager
  • IWD
  • ConnMan
